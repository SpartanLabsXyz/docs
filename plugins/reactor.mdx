Reactor is a real-time event stream plugin for OpenClaw. It connects to live Polymarket settlement data via WebSocket — trades are visible seconds before on-chain confirmation — and routes events to your agent or skills.

Instead of running skills on a timer and hoping you catch something, reactor fires the moment something happens.

## How it works

```
Real-time settlement feed → reactor plugin → dispatch
                                              ├── autoReact: execute skill immediately (no LLM)
                                              └── passive: inject events into agent context (LLM decides)
```

1. **Connect** — Plugin maintains a persistent WebSocket to a real-time Polymarket settlement feed (mempool-level data)
2. **Filter** — Events filtered server-side and client-side against your watchlist (specific wallets, minimum trade sizes)
3. **Dispatch** — Either run a skill immediately (autoReact) or buffer events for your agent to reason about (passive)

## What reactor enables

Reactor provides a raw event stream. What your agent does with it depends on your skills and strategy.

### Whale copytrading

Track specific wallets and copy their trades as they happen. In autoReact mode, the copytrading skill fires within seconds of a whale's trade — before on-chain confirmation.

### Flow-based signals

Any skill can consume reactor events as an input signal. A whale dumping $50K on "NO" is useful context for momentum, sentiment, or research-driven strategies — not just copytrading.

### Smart entry timing

If your agent already wants to trade a market, reactor events can provide confirmation. "I want to buy YES on X" + "whale just bought $20K YES on X" = higher conviction entry.

### Market discovery

Whales trading on markets your agent hasn't seen yet is a discovery signal. Reactor can surface new opportunities that your other skills can evaluate.

### Risk alerts

Detect when multiple tracked wallets exit the same market. That's a risk signal your agent should know about, even if it doesn't copy their trades.

## Dispatch modes

### Passive (default)

Events buffer in memory and get injected into your agent's context at the start of each cycle:

```
Reactor: 3 whale events since last cycle
- 0xABC... (SharkyBoi) BUY $12K on "Will ETH hit $4K" @ 0.45
- 0xDEF... BUY $8K on "Trump wins popular vote" @ 0.62
- 0xABC... (SharkyBoi) SELL $5K on "Fed rate cut June" @ 0.71

Your installed skills can act on these events.
React or ignore?
```

Your agent decides what to do — invoke a skill, ask questions, or ignore.

### autoReact

Events trigger skill execution immediately, bypassing the LLM. Fastest response time, but no nuanced reasoning.

Includes a circuit breaker: 3 consecutive skill failures auto-downgrades to passive mode.

## Event data

Each settlement event includes:

| Field | Description |
|-------|-------------|
| `whale_wallet` | Taker wallet address |
| `whale_label` | Your label for this wallet (from watchlist) |
| `side` | BUY or SELL |
| `size` | Trade size in shares |
| `price` | Execution price (0.0–1.0) |
| `market_title` | Human-readable market name |
| `market_slug` | Polymarket URL slug |
| `market_id` | Condition ID (unique market identifier) |
| `timestamp` | When the trade was detected (pre-confirmation) |

## Commands

| Command | Description |
|---------|-------------|
| `/reactor status` | Connection state, event count, dispatch mode, watchlist size |
| `/reactor pause` | Pause event processing (WS stays connected) |
| `/reactor resume` | Resume processing (also clears autoReact circuit breaker) |
| `/reactor events [n]` | Show last n events (default 10) |

## Config

| Option | Default | Description |
|--------|---------|-------------|
| `dispatch_mode` | `passive` | `passive` or `autoReact` |
| `autoReact_skill` | `polymarket-copytrading` | Skill to invoke in autoReact mode |
| `autoReact_max_usd` | `50` | Max trade size per autoReact execution |
| `buffer_max` | `100` | Max events buffered for passive injection |

## Requirements
